# Report Structure (PFE)

1. Introduction
   - Context and objectives
   - Scope
2. Requirements Analysis
   - Functional requirements
   - Non-functional requirements
3. System Design
   - Global architecture
   - Data model (MongoDB)
   - Topic design (MQTT)
4. Implementation
   - Backend (Express, Mongoose, MQTT)
   - Frontend (React, charts, auth)
   - Firmware (ESP32 simulation)
   - DevOps (Docker Compose)
5. Testing and Validation
   - API tests
   - Integration tests (MQTT -> DB -> UI)
   - Demo scenarios
6. Results and Discussion
   - Achievements
   - Limits
   - Improvements
7. Conclusion and Future Work
8. Appendices
   - Postman collection
   - Commands and scripts
